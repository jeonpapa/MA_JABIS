# openai (gpt-5)

1) Job-to-be-done
- (a)가 최우선. 이유: MA 실무의 산출물은 “협상 상한/제안가의 근거 패킷”이며, HIRA가 신뢰하는 출처·조정 로직·가정치가 모두 감사 가능해야 함.
- (b)는 비교 관점의 보조 기능(동일 적응증 경쟁 품목 포함 시 설득력↑).
- (c)는 A8 비교의 “적격국/적응증” 판정에 꼭 필요하나 단독 JTBD는 아님.
- (d)는 운영 효율을 높이는 백그라운드 기능.
결론: (a) 중심, (b)(c)는 (a)의 품질을 높이는 모듈, (d)는 유지관리 자동화.

2) 출력 단위
- 주: 파일 다운로드(xlsx/PDF). 심평원 제출·내부 결재에 필요하며, 로직·근거 링크·타임스탬프 포함 “감사용 워크북”이 핵심 산출물.
- 보조: 화면 대시보드. 목적은 작성 전·후 자체 점검(완전성, 출처 신뢰도, 매칭 정확도, 환율/공장도/마진 적용 여부)과 이슈 플래깅.

3) 제품 매칭 문제(A8 계산 단위 UX)
- 기준 정하기: “KR 기준 SKU(제형/강도/포장)”를 사용자가 1개 지정.
- 자동 정규화 규칙(가중치·신뢰도 배지 노출):
  1) Exact match: 동일 강도·포장(최고 신뢰).
  2) Strength-equivalent: 강도 차이 보정(예: 50mg×2 ↔ 100mg×1), 포장수량 스케일.
  3) Fallback per-mg 가격: 병용/다회투여 제형에서 최후 수단(추측: HIRA는 가급적 동등 포장 비교 선호).
- 사용자 승인 플로우: 자동 제안→사용자 수동 선택/수정→매핑 테이블 저장·버전관리. 각 매핑에 근거 메모(라벨링, SPC, 실제 임상 사용 단위) 필드 제공.

4) 가격+허가 통합 여부
- 한 화면 요약이 유리. “협상 캔버스”에 국가별 조정가+급여상태+적응증 매칭(해당/부분/미해당)과 제외 사유 표시. 상세 근거는 탭(Price calc, HTA/Label, Source logs)로 분리. 이렇게 해야 A8 대상국 필터링이 즉시 가능하고, 감사 시 드릴다운 가능.

5) 전체 UX 흐름
- (B) Workflow-centric이 대시보드 성격에 가장 적합. 흐름: 협상안 생성→KR 기준 SKU/적응증 선택→자동 수집/매칭→조정가 산출→자체 점검체크리스트 통과→xlsx/PDF 익스포트.
- (A) Drug-centric은 제품 단일 “dossier 1장” 뷰로 보조.
- (C) Monitoring은 워치리스트로 백그라운드 알림(변동 시 협상안에 영향도 표시).

추가 제안(가장 먼저 내릴 단 하나의 결정)
- “주 산출물=심평원 제출용 ‘증빙가능 계산서’ xlsx”를 제품의 북스타로 지정. 이 워크북 구조(시트: Source Raw, Adjustment Logic, A8 Summary, HTA Matrix, Assumptions, Audit Log)를 먼저 고정하고, 화면은 이를 작성·검증·버전관리하기 위한 수단으로 설계. 불확실: per-mg Fallback 허용 범위는 내부 규정/선례 확인 필요.
